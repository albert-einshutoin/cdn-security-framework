# やった方がいいこと・やらなくてよいこと

このフレームワークを正しく使うための「やるべきこと」と「やらなくてよいこと」の一覧です。

---

## やった方がいいこと

### 運用・開発フロー

| すること | 理由 |
|----------|------|
| **ポリシーだけ編集する** | `policy/security.yml`（または `policy/base.yml`）が唯一の正。ここを変えてビルドし直す。 |
| **変更後にビルドする** | ポリシーを編集したら `npx cdn-security build` または `npm run build` を実行し、`dist/edge/` と `dist/infra/` を再生成する。 |
| **init で始める** | 初回は `npx cdn-security init` でプラットフォーム・プロファイルを選び、雛形のポリシーを生成する。 |
| **ポリシーを検証する** | 編集後に `npm run lint:policy` でスキーマ検証する。CI でも実行する。 |
| **ビルド後にテストする** | `npm run build && npm run test:runtime` で生成コードの動作を確認する。 |
| **デプロイでトークンを設定する** | `/admin` 等の保護用に、Terraform / Wrangler 等で `EDGE_ADMIN_TOKEN` をシークレットとして設定する。 |
| **生成物はデプロイに使う** | `dist/edge/` を CloudFront Function / Lambda@Edge / Workers に、`dist/infra/*.tf.json` を Terraform でデプロイする。 |

### 設計・責務分担

| すること | 理由 |
|----------|------|
| **Edge は「前段フィルタ」と割り切る** | 軽量な正規化・遮断・ヘッダー付与は Edge。レート制限・OWASP・Bot は WAF に任せる。 |
| **業務・規制要件は別レイヤーで満たす** | 監査ログ、データ保護、業界規制（金融・HR 等）はアプリ・WAF・ログ基盤で対応する。 |
| **dist/ は CI で再生成して検証する** | ポリシーと生成物のドリフト検知が必要なら、CI 内で `npm run build` して diff を取る。`dist/` はコミットしない。 |

## やらなくてよいこと（やらない方がよいこと）

### 編集・手作業

| しないこと | 理由 |
|------------|------|
| **dist/edge/*.js や dist/edge/cloudflare/index.ts を手で編集しない** | ビルドで上書きされる。変更はポリシーに反映してからビルドする。 |
| **templates/ をユーザーとして触らない** | テンプレートは CLI の内部資産。カスタムはフォークや PR で対応。 |
| **CFG やランタイム設定を手動で同期しない** | ポリシーから自動注入されるため不要。 |
| **dist/ をリポジトリにコミットしない** | `.gitignore` 対象。各自が `npm run build` で生成する。 |

### 責務・スコープ

| このフレームに期待しないこと | 理由 |
|------------------------------|------|
| **高度な Bot 検知を Edge でやること** | Bot Management・レート制限は WAF / CDN の責務。 |
| **DB 内部の不正や業務ロジック破壊の防止** | アプリケーション層で対応する。 |
| **規制・監査の証跡をこのツールだけで満たすこと** | 監査ログ・コンプライアンスは別システムで設計する。 |
| **業種別の「金融向け」「HR向け」テンプレを標準で持つこと** | 汎用の Strict / Balanced / Permissive をベースに、自社でポリシーを調整する。 |

---

## 一覧の要約

- **やる**: ポリシー編集 → ビルド → lint/test → 生成物をデプロイ。Edge は前段に限定し、業務・規制は他レイヤーで対応。
- **やらない**: 生成コードの手編集、テンプレの直接編集、dist のコミット、このツールに Bot/DB/監査の全責任を担わせること。

## ギャップ状況

**2026-02-08 時点**で、ここに記載していた実装ギャップは解消済みです。

- Cloudflare Workers でも、policy 生成で JWT / Signed URL / Origin auth gate を利用可能。
- 標準 CI に、コミット済み golden 生成物とのドリフト検知を追加。
- コンパイラの単体テストを、compile コアと infra 出力まで拡張。
- `firewall.waf.ja3_fingerprints` による JA3 ブロックルール生成に対応。

※ 業種別テンプレ・監査・規制対応は意図的なスコープ外であり、実装ギャップではありません。
